## 火苗（最优先，面试必问，直接影响编码质量）

### 1. 响应式原理（Proxy）
**为什么重要：** 理解了 Proxy，你就知道 ref/reactive 的底层区别、为什么解构丢失响应式、为什么 Vue 3 不需要 `$set`

| 要搞懂的点 | 说明 |
|-----------|------|
| Proxy 的 `get` / `set` 拦截 | Vue 3 如何追踪依赖 |
| ref 为什么需要 `.value` | 原始值无法被 Proxy 代理，包一层对象 |
| reactive 解构丢失响应式 | 解构后变成原始值，Proxy 管不到了 |
| `toRefs` / `toRef` | 解构时保留响应式连接的方案 |
| 为什么不需要 `$set` / `$delete` | Proxy 能拦截新增/删除属性，defineProperty 不行 |
| `markRaw` | 跳过某些数据的响应式代理，性能优化 |

### 2. Composition API 的设计权衡
**为什么重要：** 不是"新的比旧的好"，而是在合适的场景用合适的方案

| 问题 | 本质 |
|------|------|
| 什么时候用 Composition API | 跨组件逻辑复用、复杂组件拆分 |
| 什么时候用 Options API | 简单组件、模板代码更直观 |
| 对比 React Hooks | 为什么 Vue 的 `watchEffect` 不需要依赖数组？→ Proxy 自动追踪 |
| Composables 命名规范 | `useXxx` 约定，纯函数式组合 |

### 3. keep-alive / Teleport / Suspense
**为什么重要：** 实际项目天天用

| 特性 | 场景 |
|------|------|
| `KeepAlive` | 路由切换时保留表格搜索状态、Tab 切换、避免重复请求 |
| `Teleport` | 弹窗/模态框/下拉菜单溢出父容器定位问题 |
| `Suspense` + 异步组件 | 异步加载组件时的 loading 状态、fallback 管理 |

## 余烬（面试加分，但不影响日常编码质量）

### 4. 虚拟 DOM & diff 算法
| 要搞懂的点 | 说明 |
|-----------|------|
| 为什么需要虚拟 DOM | 直接操作 DOM 太慢，JS 层面做 diff 后再批量更新 |
| `:key` 的原理 | 对比 VNode 时通过 key 判断是移动还是重建 |
| Vue 3 的 PatchFlag | 编译时标记动态节点，diff 时跳过静态内容 → 性能提升 |

### 5. 编译优化
| 优化手段 | 效果 |
|---------|------|
| 静态提升 | 不变的 VNode 只创建一次 |
| Tree-shaking | 没用到的 API 不打包 |
| PatchFlag | diff 时只对比变化的部分 |
